# Education Archive 아카이브 - 조회수 자동 집계 서버

미리캔버스 대신 이 사이트를 운영할 경우, 클릭할 때마다 조회수가 자동으로 쌓이는 아주 가벼운 서버입니다.
파이썬 표준 라이브러리만 사용해서 별도 설치(pip install 등) 없이 바로 실행됩니다.

## 폴더 구성

```
archive_site_server/
├── server.py         # 서버 본체 (정적 파일 서빙 + 조회수 API)
├── seed_data.json     # 미리캔버스에서 이미 확인한 초기 조회수 (95개 항목)
└── public/             # 실제 화면 (index.html, 상품별 페이지, style.css)
```

## 실행 방법

```bash
cd archive_site_server
python3 server.py
```

브라우저에서 `http://localhost:8000` 접속하면 됩니다.
서버를 처음 켤 때 `clicks.db`(SQLite 파일)가 자동으로 생성되면서,
`seed_data.json`에 있는 기존 조회수 95건이 그대로 입력됩니다.
이후부터는 실제로 버튼을 클릭할 때마다 그 값 위에 1씩 더해집니다.

**주의:** `clicks.db`가 한번 생성된 이후에는 seed_data.json을 다시 읽지 않습니다.
데이터를 초기화하고 싶으면 `clicks.db` 파일을 지우고 서버를 다시 실행하세요.

## 조회수는 어떻게 기록되나요?

- 각 항목 카드의 "미리캔버스로 보기" 버튼을 누르면 → `miricanvas_views` +1
- "▶ 유튜브" 버튼을 누르면 → `youtube_views` +1
- 페이지를 열 때마다 최신 조회수를 `/api/counts`에서 불러와 화면에 표시합니다.
- 허브 페이지(index.html)의 각 카드에는 `/api/summary`로 계산한 상품별 합계가 표시됩니다.

## API 목록

| Method | 경로 | 설명 |
|---|---|---|
| GET | `/api/counts` | 전체 항목별 조회수 (URL을 key로 하는 JSON) |
| GET | `/api/summary` | 상품(대분류)별 합계 조회수 |
| POST | `/api/click` | `{ "url": "...", "type": "miricanvas" \| "youtube", "name": "...", "product": "..." }` 를 보내면 해당 항목 조회수 +1 |

## 실제 사내에서 여러 명이 쓰게 하려면

지금 방식은 **로컬 테스트용**이라, 사용자가 각자 컴퓨터에서 `python3 server.py`를 실행하면 조회수가 서로 다르게 기록됩니다.
사내에서 모두가 같은 조회수를 보게 하려면:

1. 사내 서버(또는 클라우드 VM) 한 대에 이 폴더를 올리고
2. `python3 server.py`를 **계속 켜져 있는 상태**로 실행 (예: `nohup python3 server.py &`, 또는 systemd/pm2 같은 프로세스 매니저 사용)
3. 사내 구성원들이 그 서버의 주소(`http://사내서버IP:8000`)로 접속

이 부분은 실제 배포 환경(사내망 구조, 방화벽, 도메인 여부 등)에 따라 설정이 달라지므로,
사내 서버 정보를 알려주시면 그에 맞는 배포 스크립트도 추가로 만들어드릴 수 있습니다.

## 참고: 시드 데이터 출처

나머지 76개 항목은 같은 시트 안에서 이름이 동일한 행을 찾아 매칭했습니다).
